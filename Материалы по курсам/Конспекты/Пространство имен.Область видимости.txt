
   Пространство имен (namespace) - это некая область, состоящая из некоторого множества ссылок от всех имен 
до объектов. В пределах пространства каждое имя уникально.
   Оно состоит из сопоставления имен переменных, функций, классов и реальных объектов в оперативной памяти.

Так выражение присваивания создает имя (символическое имя), которое может использоваться для ссылки на объект:
Пример: 
     x = 'foo' 

# x - символическое имя 
# foo - строковый объект, на который ссылается (x)

   Пространства имен включает следущее:
1. Встроенное (имена всех встроенных, предустановленных объектов python)
2. Глобальное (имена на уровне основной программы)
3. Объемлющее (имена на уровне отдельной функции, если внутри нее есть еще вложенные функции)
4. Локальное (имена на уровне отдельной функции)

   Область видимости (scope, по русски - масштаб, область) - область, в которой данное имя 
обладает значением. Т.е. область видимости определяет контекст имени (переменной, функции и тд.),
в рамках которого его можно использовать.

   Когда говорим о пространстве имен, то подразумевается некоторая область с неким колличеством ссылок 
от имен к объектам. Когда же говорим о области видимости, подразумевается в первую очередь 
возможность использования имени в конкретной области, месте.

   Интерпретатор определяет эту область, основываясь на следующем:
1. Где располагается определение имени
2. Из какого места в коде на него ссылаются
   
   А потом следует правилу LEGB(правила областей видимости), которое суть порядок поиска:
   1. Локальная. 
Если вы ссылаетесь на x внутри функции, то интерпретатор сначала ищет его в самой внутренней области, 
локальной для этой функции. 
   2. Объемлющая.
Еслиx не находится в локальной области, но появляется в функции, располагающейся внутри другой функции, 
то интерпретатор ищет его в области видимости объемлющей функции. 
   3. Глобальная.
Если ни один из вышеуказанных вариантов не принес результатов, то интерпретатор продолжит поиск 
в глобальной области видимости. 
   4. Встроенная.
Если интерпретатор не может найти x где-либо еще, то он направляет поиски во встроенную область видимости. 

   Таким образом локальная переменная (которая находится в функции) допустим, x = 100, 
не будет иметь значение в глобальном пространстве.
